# Reports

BeGuardit generates reports in four formats. The CLI produces JSON and HTML locally; the API worker generates HTML, PDF, and SARIF on demand.

## JSON Canonical

The primary output format. Machine-readable, tamper-evident, and used for API uploads.

**Produced by:** CLI (`beguardit start`)

```json
{
  "schema_version": "1.0",
  "session_id": "a1b2c3d4-...",
  "generated_at": "2026-03-18T12:00:00.000Z",
  "hostname": "prod-server-01",
  "os_info": { ... },
  "scan_config": { "profile": "deep", "categories": ["cyber", "ai"] },
  "summary": {
    "total_findings": 16,
    "by_severity": { "critical": 2, "high": 4, "medium": 7, "low": 3 },
    "total_assets": 42,
    "total_evidence": 28,
    "total_attack_paths": 3
  },
  "assets": [ ... ],
  "findings": [ ... ],
  "evidence": [ ... ],
  "attack_paths": [ ... ],
  "integrity": {
    "algorithm": "sha256",
    "hash": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
  }
}
```

### Integrity Verification

The `integrity.hash` is a SHA-256 digest of the entire report body (excluding the integrity field itself). This enables tamper detection:

```bash
# Extract the report body (without integrity field) and verify
jq 'del(.integrity)' report.json | sha256sum
```

## HTML

Self-contained HTML report with inline CSS. No external dependencies — works offline, printable, and email-friendly.

**Produced by:** CLI (with `--format html`) or API worker

Features:
- Severity summary cards with color coding
- Ordered findings list (critical first)
- Asset inventory table
- Responsive layout for all screen sizes

```bash
beguardit start --format html
# Output: ./beguardit-report.html
```

## PDF

Generated by the API worker using WeasyPrint. Renders the HTML report to PDF for formal compliance deliverables.

**Produced by:** API worker (POST /api/v1/reports)

```bash
# Via the portal: Assessment Detail → Generate Report → PDF
# Via the API:
curl -X POST https://api.beguardit.com/api/v1/reports/ \
  -H 'Content-Type: application/json' \
  -d '{"session_id": "...", "format": "pdf"}'
```

## SARIF v2.1.0

[Static Analysis Results Interchange Format](https://sarifweb.azurewebsites.net/) for integration with:

- **GitHub Code Scanning** — upload as a code scanning alert
- **Azure DevOps** — native SARIF viewer
- **VS Code** — SARIF Viewer extension

**Produced by:** API worker

```bash
# Generate SARIF report
curl -X POST https://api.beguardit.com/api/v1/reports/ \
  -H 'Content-Type: application/json' \
  -d '{"session_id": "...", "format": "sarif"}'
```

### SARIF Severity Mapping

| BeGuardit | SARIF Level |
|-----------|-------------|
| critical | error |
| high | error |
| medium | warning |
| low | note |
| info | none |

### GitHub Code Scanning Integration

Upload a SARIF report to GitHub:

```bash
# Download the SARIF file
curl -O https://api.beguardit.com/api/v1/reports/{id}/download

# Upload to GitHub Code Scanning
gh api /repos/{owner}/{repo}/code-scanning/sarifs \
  -f "sarif=$(gzip -c report.sarif.json | base64)" \
  -f "ref=refs/heads/main"
```

## Report Generation via API

The API uses an async job queue (ARQ + Redis) for report generation:

1. **POST /api/v1/reports/** — creates a job, returns `job_id`
2. **GET /api/v1/reports/{id}** — poll for status (`queued` → `processing` → `completed`)
3. **GET /api/v1/reports/{id}/download** — download the file when complete

The portal automates this with a 10-second polling interval.
